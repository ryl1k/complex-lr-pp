# Технічне завдання до ЛР-4 з Прикладного Програмувння
- Варіант 8, тема "Транспорт"

**Загальні відомості**
- Назва програми: «Пасажирський потяг»
- Мета розробки: створити консольну програму для моделювання складу пасажирського поїзда, управління вагонами, виконання базових операцій (додавання, видалення, сортування, пошук, підрахунок), а також для збереження і завантаження інформації з файлу.

**Призначення розробки**
- Програма призначена для демонстрації принципів об’єктно-орієнтованого програмування (класи, інкапсуляція, успадкування, поліморфізм), а також для навчання роботі зі структурами даних і файлами у мові Java.
- Користувач може створювати поїзд, редагувати його склад, переглядати інформацію про вагони, виконувати пошук і обчислення, зберігати та завантажувати дані.

# Функціональні вимоги
**Програма повинна реалізовувати такі функції:**
- створення нового поїзда;
- завантаження поїзда з файлу;
- виведення інформації про склад поїзда;
- додавання нового вагона;
- редагування параметрів вагона;
- видалення вагона;
- сортування вагонів за рівнем комфортності;
- підрахунок загальної кількості пасажирів і багажу;
- пошук вагонів за заданим діапазоном кількості пасажирів;
- збереження змін у файл;
- відображення довідки та вихід із програми.

# Вимоги до структури програми
- Програма має бути побудована з використанням принципів ООП.

**Основні класи:**
- Wagon: базовий клас вагона, що містить основні параметри (id, тип, кількість пасажирів, багаж, рівень комфорту).
- PassengerWagon: підклас Wagon із додатковими властивостями (наявність WiFi, кондиціонера).
- LuxuryWagon: підклас PassengerWagon, який представляє вагон підвищеного класу.
- Train: клас, що агрегує об’єкти Wagon і містить методи для керування складом поїзда.
- FileService: клас для зчитування та запису інформації про поїзд у файл (композиційний зв’язок із Train).
- TrainService: клас для виконання розрахунків (сортування, підрахунки, фільтрація).
- ConsoleUI: клас, що реалізує взаємодію з користувачем через консольне меню.

# Вимоги до реалізації
- Мова програмування: Java.
- Тип програми: консольна.

**Пакетна структура:**
- model (класи моделей: Wagon, PassengerWagon, LuxuryWagon, Train);
- service (логіка обробки даних: FileService, TrainService);
- ui (користувацький інтерфейс: ConsoleUI).

Передбачити використання модифікаторів доступу, геттерів/сеттерів, поліморфізму, колекцій (List), обробку виключень при роботі з файлами.
**Дані про поїзд зберігаються у текстовому файлі у форматі CSV або JSON.**

# Вимоги до інтерфейсу користувача
Програма повинна мати консольне меню, яке дозволяє вибрати одну з дій:

- Завантажити поїзд з файлу
- Показати всі вагони
- Додати вагон
- Редагувати вагон
- Видалити вагон
- Порахувати кількість пасажирів і багажу
- Сортувати вагони за комфортом
- Знайти вагони за діапазоном пасажирів
- Зберегти у файл
- Вихід

# Вхідні та вихідні дані
- Вхідні дані: інформація про вагони (ідентифікатор, тип, кількість пасажирів, багаж, рівень комфорту, додаткові параметри).
- Вихідні дані: звіти про склад поїзда, підраховані значення, результати пошуку, повідомлення про виконання операцій.

# Вимоги до тестування
Перевірити коректність:
- створення та додавання вагонів;
- завантаження та збереження у файл;
- роботи пошуку та сортування;
- обчислення сумарних значень;
- обробки помилкових даних (порожній файл, неправильний формат, відсутність вагонів).

# Очікуваний результат
Програма повинна забезпечувати коректне управління складом пасажирського поїзда, підтримувати роботу з файлами, виконувати підрахунки та відображати результати в зручній формі.
Структура коду має бути розділена за пакетами та відповідати принципам ООП.
